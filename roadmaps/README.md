# 🗺️ Roadmaps

Step-by-step learning paths so you always know what to study next. Use these as a spine, then pull depth from [books/](../books/), [papers/](../papers/), and [courses/](../courses/).

## 🌱 0 → 1: Become a Software Engineer

1. **Programming fundamentals** — pick one language (Python or JavaScript) and get fluent. → [CS50](../courses/)
2. **Data structures & algorithms** — arrays, hashmaps, trees, graphs, big-O. → [The Pragmatic Programmer](../books/)
3. **Version control & tooling** — git, the command line, debugging. → [Missing Semester](../courses/)
4. **How computers work** — memory, the stack, networking basics. → [Code](../books/) · [CS:APP](../books/)
5. **Build & ship a project** — a small full-stack app, deployed. → repeat.

## 🚀 1 → Senior: Deepen and broaden

1. **Software design** — clean code, refactoring, modularity. → [A Philosophy of Software Design](../books/), [Refactoring](../books/)
2. **Data systems** — databases, caching, queues, consistency. → [Designing Data-Intensive Applications](../books/)
3. **System design** — scaling, trade-offs, architecture. → [System Design Primer](../courses/), [System Design Interview](../books/)
4. **Distributed systems** — consensus, replication, fault tolerance. → [MIT 6.824](../courses/), [papers/](../papers/)
5. **Production maturity** — reliability, observability, on-call. → [SRE](../books/), [Release It!](../books/)
6. **Testing & quality** — unit/integration/e2e, CI/CD. → practice in your project.

## 🤖 The AI-Assisted Engineer (2025+)

1. **Foundations** — how LLMs and transformers work. → [Attention Is All You Need](../papers/)
2. **Working with agents** — prompting, tool use, the agentic loop. → [ReAct](../papers/), [The New SDLC With Vibe Coding](../whitepapers/ai-agentic-engineering/)
3. **Building AI products** — RAG, evals, guardrails. → [AI Engineering](../books/)
4. **Agentic engineering practice** — spec-driven development, review discipline, verification. → [The New SDLC With Vibe Coding](../whitepapers/ai-agentic-engineering/)

---

> 📌 **How to use a roadmap:** don't binge. Take one numbered step at a time, build something with it, then move on. Depth beats checklist-completion.
