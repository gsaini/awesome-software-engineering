# 🤝 Contributing

Thanks for helping make this library better! Contributions of all sizes are welcome — a missing classic, a better explanation, a fixed link.

## What belongs here

High-signal, durable learning material for software engineering:

- **Whitepapers** — industry PDFs (AI, distributed systems, databases, infra).
- **Books** — books worth recommending to a peer.
- **Papers** — foundational or influential academic papers.
- **Courses** — structured courses, lectures, video series.
- **Roadmaps & Resources** — learning paths, blogs, newsletters, podcasts.

**Quality over quantity.** If you wouldn't personally recommend it to a colleague, leave it out.

## How to add a resource

1. **Find the right folder** and open its `README.md`.
2. **Add a row** to the matching table with:
   - The title, linked to the **official / canonical source** (publisher, arXiv, lab page — never a pirated copy).
   - Author(s) or provider.
   - A **level badge**: 🟢 Beginner · 🟡 Intermediate · 🔴 Advanced.
   - A **one-line "why it matters"** — concrete, not marketing copy.
3. For **whitepaper PDFs**, drop the file in the matching `whitepapers/<domain>/` folder using a `kebab-case-title.pdf` filename, and add a table row.
4. If the main [README.md](README.md) has a relevant summary table, update it too.

## Style

- Keep annotations short and honest (one line).
- Use the level badges consistently.
- Use `kebab-case` for filenames.
- One logical change per pull request.

## Content & licensing rules

- **Do not commit pirated or paywalled content.** Link to where people can legitimately buy or access it.
- Only add PDFs that are freely and legally distributable (e.g. openly published whitepapers).
- This repo is **non-commercial and for learning only** — see the [disclaimer](README.md#-disclaimer). All credit belongs to original authors.

## Opening a pull request

1. Fork and create a branch (`add/<short-name>`).
2. Make your change.
3. Open a PR with a clear title and a sentence on why the resource is worth adding.

Not sure if something fits? Open an issue first and ask — happy to discuss.
