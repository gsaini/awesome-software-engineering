# 📗 Books

A curated, annotated reading list. Every book here earns its place. Levels: 🟢 Beginner · 🟡 Intermediate · 🔴 Advanced.

> 💡 Please **buy books from official sources** to support the authors. Links point to publishers/retailers, not pirated copies. See the [disclaimer](../README.md#-disclaimer).

## 🧱 Foundations & Computer Science

| Book | Author | Level | Why it matters |
| ---- | ------ | ----- | -------------- |
| Structure and Interpretation of Computer Programs (SICP) | Abelson & Sussman | 🔴 | Rewires how you think about programs and abstraction. Free online. |
| The Pragmatic Programmer (20th Anniversary Ed.) | Hunt & Thomas | 🟢 | The mindset and habits of effective engineers. |
| Code: The Hidden Language of Computer Hardware and Software | Charles Petzold | 🟢 | From relays to CPUs — how computers actually work. |
| Computer Systems: A Programmer's Perspective (CS:APP) | Bryant & O'Hallaron | 🔴 | Memory, caches, linking, concurrency from the machine up. |

## 🏗️ System Design & Architecture

| Book | Author | Level | Why it matters |
| ---- | ------ | ----- | -------------- |
| Designing Data-Intensive Applications | Martin Kleppmann | 🟡 | The single best book on modern data systems. Read it twice. |
| System Design Interview (Vol. 1 & 2) | Alex Xu | 🟡 | Practical, diagram-driven prep for design rounds. |
| Software Architecture: The Hard Parts | Ford, Richards, et al. | 🔴 | Trade-off analysis for distributed architectures. |
| Fundamentals of Software Architecture | Richards & Ford | 🟡 | Vocabulary and characteristics of good architecture. |

## 🛠️ Software Craft & Practices

| Book | Author | Level | Why it matters |
| ---- | ------ | ----- | -------------- |
| Clean Code | Robert C. Martin | 🟢 | Controversial but foundational vocabulary for readable code. |
| Refactoring (2nd Ed.) | Martin Fowler | 🟡 | A catalog of safe, incremental code improvements. |
| Working Effectively with Legacy Code | Michael Feathers | 🔴 | How to change code you're afraid to touch. |
| The Mythical Man-Month | Fred Brooks | 🟢 | Timeless truths about software projects and teams. |
| A Philosophy of Software Design | John Ousterhout | 🟡 | Deep modules, complexity, and design intuition. |
